# AI 常见错误模式记录

> 本文档记录了 AI 在辅助论文写作过程中犯过的典型错误，供后续对话参考，避免重复犯错。

---

## 1. 夸大系统功能

**错误描述**：AI 倾向于将系统描述得比实际更强大、更完整。

**典型案例**：
- 在第 2 章控制策略设计中，AI 写道"系统可通过 CAN 总线将 LLM 决策结果下发至指定从节点执行，实现从语音指令到执行器动作的端到端智能控制链路"。实际上，当前版本的 AI 领航模式**仅支持信息查询与建议生成**，不具备直接控制执行器的能力。

**纠正规则**：
- 描述系统功能时，必须严格基于代码中实际实现的功能，不要推测或"展望"未来可能的功能。
- AI（DeepSeek）当前只能给出**文本建议和分析报告**，通过页面展示和语音播放呈现给用户，不能直接操控执行器。
- 如果代码中没有实现某个功能（如 AI 直接下发 CAN 控制命令），就不能在论文中声称系统具备该功能。

## 2. 引用滥用

**错误描述**：AI 在不恰当的地方添加学术引用，引用的内容并非该文献的核心贡献。

**典型案例**：
- 引用 `@niveditha2018can` 来支持"CAN 总线支持 127 个从节点"的说法。实际上，127 个节点是 CAN 2.0A 协议标准本身定义的（7-bit 节点 ID），不需要引用某篇特定论文来证明。
- 引用某篇论文来支持一个通用技术事实（如"I2C 总线采用 SDA 和 SCL 两根信号线"），而该事实属于公共知识。

**纠正规则**：
- **公共知识不需要引用**：CAN 帧格式、I2C 协议基础、PID 算法原理等属于教科书级别的公共知识，不需要引用特定论文。
- **引用用于以下场景**：
  - 引入一个研究领域或技术方向（综述类文献）
  - 支持某个具体的技术选型或设计方案
  - 对比现有研究的不足
  - 引用数据手册中的具体参数
- **不要为了"看起来学术"而堆砌引用**。每条引用都应有明确的语义目的。

## 3. 图表可读性不足

**错误描述**：AI 生成的 Mermaid 图表在 A4 纸上打印后文字过小、无法阅读。

**典型案例**：
- 系统架构图使用 LR 布局，viewBox 宽度达到 1921px，在 A4 纸上缩放后文字仅约 5px，完全无法阅读。
- 层级架构图使用 LR 布局将所有模块水平排列，viewBox 宽度超过 3000px。
- CAN 帧格式图使用 LR 布局，viewBox 宽度 1362px，缩放后文字约 6px。

**纠正规则**：
- **A4 纸有效宽度约 500px（72 DPI）**。SVG 的 viewBox 宽度应控制在 **800px 以内**，以确保 16px 基准字体缩放后不低于 10px。
- **图表宽度自检公式**：`缩放后字号 = 16 × (500 / viewBox宽度)`，结果应 ≥ 10px。
- **优先使用 TD（自上而下）布局**，避免 LR 布局导致宽度过大。
- **节点数量控制在 5-10 个**，超过时应拆分或简化。
- **对于结构化数据（如帧格式、参数表），优先使用 Markdown 表格**而非流程图。
- **流程图应传达"流程"和"逻辑关系"**，不要用流程图来展示静态的结构化信息。

## 4. 工具限制记录

**以下工具/功能在当前环境下不可用，不要尝试使用**：

| 工具/功能 | 不可用原因 |
|:---|:---|
| 读取 PNG/JPG 图片 | **绝对不可用** — 详见下方说明 |
| DuckDuckGo 搜索 | 代理 SSL 握手失败 |
| `ddgs` CLI | 代理 SSL 握手失败 |

### ⚠️ 关于读取 PNG/JPG 图片的重要说明

**问题本质**：AI 工具链中的 Read 工具无法解析 PNG/JPG 等位图文件的像素内容。Read 工具只能读取纯文本文件（如 .md、.py、.json、.mmd 等），对于二进制图片文件，它会返回乱码或错误信息，**无法提取图片中的任何视觉信息**。

**常见错误场景**：
- AI 试图通过 Read 工具读取 PNG 图片来"查看"图表内容
- AI 试图通过读取 PNG 文件来确认图表是否正确渲染
- AI 试图从 PNG 图片中提取文字或数据信息

**正确做法**：
1. **查看图表内容** → 读取对应的 `.mmd`（Mermaid 源文件）或 `.svg`（矢量图，可读取文本内容）
2. **确认图表渲染效果** → 请用户手动打开 PNG 文件查看，或使用 `mcp__agent-browser__browser_screenshot` 工具截图
3. **修改图表** → 编辑 `.mmd` 源文件，然后重新生成 PNG

**示例**：
```bash
# ❌ 错误：试图读取 PNG 图片
Read("Docs/images/architecture.png")  # 无法解析位图内容

# ✅ 正确：读取 Mermaid 源文件
Read("Docs/images/architecture.mmd")  # 可读取文本内容并修改

# ✅ 正确：读取 SVG 文件（SVG 是 XML 格式，可读取文本）
Read("Docs/images/architecture.svg")  # 可读取 XML 文本内容
```

## 5. 术语一致性

**错误描述**：同一概念在不同章节使用不同名称。

**纠正规则**：
- 全文统一使用同一术语，首次出现时给出英文全称或缩写。
- 常见易混术语：
  - "迟滞控制" vs "滞环控制" → 统一使用"迟滞控制（Hysteresis Control）"
  - "主节点" vs "主机" → 统一使用"主节点"
  - "从节点" vs "从机" → 统一使用"从节点"
