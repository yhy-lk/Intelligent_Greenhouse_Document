# 第6章 总结与展望

## 6.1 工作总结

本课题围绕"分布式多节点架构 + 大语言模型智能决策"的核心理念，设计并实现了一种基于 ESP32 与 STM32 的分布式智能温室控制系统。系统采用"ESP32 主节点 + 多个 STM32 从节点"的分层架构，主从节点之间通过 CAN 总线实现可靠的数据通信，构建了"多节点感知—集中决策—分布式执行"的智能温室控制体系。本课题完成的主要工作如下：

（1）**系统架构设计**。设计了以 ESP32-S3 为主节点、STM32F407 为从节点的分布式多节点架构。主节点承担 GUI 显示、语音交互、WiFi 联网和大语言模型推理等交互层功能，从节点承担传感器数据采集、本地自动控制和执行器驱动等控制层功能。主从节点的软件均采用分层架构设计，STM32 侧基于 Embassy 异步运行时框架按 firmware/app/bsw/service 四层组织，ESP32 侧按 App/Service/ECUAL/MCAL/BSP 五层组织，各层通过标准接口实现编译期依赖隔离。

（2）**CAN 总线多节点通信协议设计**。基于 CAN 2.0A 标准设计了适用于温室场景的多节点通信协议，采用 11-bit CAN ID 结构（4-bit 功能码 + 7-bit 节点 ID），定义了数据上报（Report）、控制命令（WriteSet）、时间同步（TimeSync）与异常告警（Alert）四类功能码，支持最多 127 个从节点的寻址与通信。

（3）**STM32 从节点控制层实现**。完成了 SHT30 温湿度传感器、BH1750 光照传感器和土壤湿度传感器的驱动开发，实现了通风风扇的离散位置式 PID 转速闭环控制，以及温度、湿度、土壤湿度和光照的迟滞控制策略。本地自动调度器以 100ms 周期独立运行，各从节点无需主节点干预即可完成本地闭环控制。

（4）**ESP32 主节点交互层实现**。基于 LVGL v9.3.0 框架设计了包含主页、总览页、控制页、自动模式页和 AI 领航页在内的图形界面系统；集成了 DeepSeek 大语言模型 API，实现了多节点传感器数据聚合注入与智能分析；结合百度语音识别与语音合成 API，构建了"语音输入—大模型推理—语音输出"的完整语音交互链路。

（5）**三种控制模式实现**。设计并实现了手动控制、自动控制和 AI 领航三种控制模式。手动模式由用户通过主节点触摸屏直接操控指定从节点的执行器；自动模式由各从节点独立运行迟滞调度器进行本地闭环控制；AI 领航模式通过语音交互驱动，DeepSeek 大语言模型综合分析多节点环境数据后给出决策建议。

## 6.2 创新点

本课题的创新点主要体现在以下几个方面：

（1）**分布式多节点控制架构**。区别于传统智能温室系统采用的单 MCU 集中式架构，本系统采用 ESP32 主节点与多个 STM32 从节点的分布式架构，通过 CAN 总线实现主从节点间的数据通信。该架构将交互层与控制层解耦，各从节点独立承担本地控制任务，单节点故障不影响其他节点运行，系统可扩展性与可靠性显著提升。

（2）**大语言模型与温室控制的深度融合**。将 DeepSeek 大语言模型集成到嵌入式温室控制系统中，通过多节点传感器数据聚合注入构建 LLM 上下文，实现了基于自然语言交互的"AI 领航"控制模式。用户可通过语音与系统对话，大模型综合分析多区域环境数据后给出决策建议，突破了传统温室系统依赖预设规则的智能化瓶颈。

（3）**多节点数字孪生状态管理**。在 ESP32 主节点设计了 SensorState 数字孪生模块，以线程安全的方式实时维护所有在线从节点的传感器读数与控制状态。该模块为 GUI 界面展示和 LLM 上下文构建提供统一的数据接口，快照读取机制避免了多任务并发访问导致的数据不一致问题。

（4）**本地闭环与远程调度相结合**。STM32 从节点的本地迟滞调度器保证了控制的实时性与独立性，ESP32 主节点的远程调度实现了全局视角的协调优化。两种控制方式通过模式切换机制有机结合，在网络中断时从节点仍可独立维持自动控制。

## 6.3 不足与展望

### 6.3.1 现有不足

尽管本课题实现了预期的系统功能，但受限于开发周期和实验条件，系统仍存在以下不足之处：

（1）传感器种类有限。当前系统仅集成了温度、湿度、光照和土壤湿度四类传感器，缺乏 CO$_2$ 浓度、pH 值和电导率（EC）等对作物生长同样重要的环境参数监测能力。

（2）AI 交互延迟较大。DeepSeek 大语言模型 API 当前采用非流式返回模式（`stream: false`），需等待完整响应生成后才能输出，导致用户从语音输入到听到回复的等待时间较长，交互体验有待优化。

（3）历史数据管理缺失。系统当前仅维护各节点的实时状态快照，未实现历史数据的存储与查询功能。趋势分析页面和系统设置页面的 GUI 骨架已由 GUI Guider 生成，但业务逻辑尚未实现。

（4）电源管理方案简略。系统直接复用开发板 USB 供电方案，未设计独立的电源管理电路，无法适应无市电供应的野外温室场景。

### 6.3.2 未来展望

针对上述不足，未来可从以下方向进行改进与扩展：

（1）**丰富传感器体系**。增加 CO$_2$ 传感器（如 MH-Z19C）、pH 传感器和 EC 传感器，实现对温室气体浓度和土壤养分的全面监测，为大语言模型提供更完整的环境数据输入。

（2）**优化 AI 交互体验**。将 DeepSeek API 切换为流式返回模式（`stream: true`），实现"边生成边播放"的流式语音输出，显著缩短用户等待时间。同时可探索将轻量级推理模型部署至 ESP32 本地，实现离线状态下的基础智能问答。

（3）**完善数据管理功能**。实现趋势页面的多节点历史数据曲线展示和对比分析，实现系统设置页面的节点管理与网络配置功能。可引入 SD 卡或外挂 Flash 作为本地数据存储介质，支持离线数据记录与导出。

（4）**引入边缘 AI 能力**。探索在 ESP32-S3 上部署 TinyML 模型，实现本地化的环境异常检测与简单决策推理，减少对云端大模型的依赖，提升系统在网络不稳定场景下的自主决策能力。

（5）**设计独立电源模块**。开发包含锂电池充放电管理、太阳能充电和低功耗模式的电源管理方案，使系统能够部署在无市电供应的野外温室环境中。

（6）**扩展多温室集群管理**。在现有单温室分布式架构的基础上，通过 WiFi/4G 网络将多个温室的 ESP32 主节点接入统一的云端管理平台，实现多温室集群的远程监控与协同管理。
