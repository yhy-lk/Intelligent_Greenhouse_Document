# 第6章 总结与展望

## 6.1 工作总结

本课题设计并实现了一种基于 ESP32 与 STM32 的分布式智能温室控制系统，完成的主要工作如下：

（1）**系统架构设计**。设计了"ESP32 主节点 + 多 STM32 从节点"的分布式架构，主从节点通过 CAN 总线通信，软件采用分层架构实现编译期依赖隔离。

（2）**控制层实现**。完成了多类传感器驱动开发，实现了风扇 PID 转速闭环控制及温湿度、光照、土壤湿度的迟滞控制策略，各从节点独立运行本地闭环控制。

（3）**交互层实现**。基于 LVGL 设计了多页面 GUI 系统，集成了 DeepSeek 大语言模型 API 和百度语音 API，实现了语音交互驱动的"AI 领航"控制模式。

## 6.2 创新点

（1）**分布式多节点控制架构**。采用 ESP32 主节点与多个 STM32 从节点的分布式架构，交互层与控制层解耦，单节点故障不影响其他节点运行，系统具备良好的可扩展性，新增从节点仅需分配唯一节点 ID 并接入 CAN 总线即可。

（2）**大语言模型与温室控制的深度融合**。将 DeepSeek 大语言模型集成到嵌入式系统中，通过多节点数据聚合注入构建 LLM 上下文，实现了基于自然语言交互的智能环境分析与建议生成，在一定程度上改善了传统温室系统完全依赖预设规则进行决策的局限性。

（3）**本地闭环与远程调度相结合**。STM32 从节点的本地迟滞调度器保证控制实时性，ESP32 主节点实现全局视角的协调优化，网络中断时从节点仍可独立维持自动控制。

## 6.3 不足与展望

### 6.3.1 现有不足

（1）传感器种类有限，缺乏 CO$_2$ 浓度、pH 值和电导率等参数监测能力。

（2）AI 交互延迟较大，当前采用非流式返回模式，等待时间较长。

（3）历史数据管理缺失，仅维护实时状态快照，未实现数据存储与查询。

### 6.3.2 未来展望

（1）**丰富传感器体系**。增加 CO$_2$、pH、EC 传感器，为大语言模型提供更完整的环境数据输入。

（2）**优化 AI 交互体验**。切换为流式返回模式，探索将轻量级模型部署至 ESP32 本地实现离线智能问答。

（3）**完善数据管理功能**。实现历史数据曲线展示与对比分析，引入 SD 卡支持离线数据记录与导出。

（4）**设计独立电源模块**。开发锂电池充放电管理和太阳能充电方案，适应野外温室场景。
