## 第2章 需求分析与总体设计（2,000-3,000 字）

### 2.1 系统需求分析（600-1,000 字）
#### 2.1.1 功能需求
- 环境数据采集：温度、湿度、光照、土壤湿度
- 执行器控制：通风风扇、水泵、加湿器、补光灯、遮阳舵机
- 三种控制模式：手动模式、自动模式、AI 领航模式
- 用户交互：触摸屏 GUI、语音交互、实时数据显示
- 数据通信：CAN 总线、WiFi 联网、云端 API 调用

#### 2.1.2 性能需求
- 传感器采样周期：≤ 1s
- 控制响应时间：≤ 100ms
- CAN 总线通信速率：1 Mbps
- 系统稳定性：7×24 小时连续运行

#### 2.1.3 非功能需求
- 低功耗设计
- 模块化与可扩展性
- 故障自检与容错机制

### 2.2 系统总体架构设计（600-1,000 字）
#### 2.2.1 分布式多节点架构设计
- **主节点（ESP32-S3）职责**：GUI 显示、语音处理、WiFi 联网、AI 推理、多节点状态管理
- **从节点（STM32F407）职责**：传感器采集、PID 控制、执行器驱动、本地自动调度
- **架构优势分析**：
  - 任务解耦：交互与控制分离，降低单节点复杂度
  - 可扩展性：CAN 总线原生支持 127 个子节点，可灵活扩展温室区域
  - 可靠性：单节点故障不影响其他节点运行
  - 实时性：STM32 从节点本地闭环控制，响应延迟 < 10ms

#### 2.2.2 CAN 总线多节点通信架构
- **拓扑结构**：ESP32 主节点 ↔ 多个 STM32 从节点（总线型拓扑）
- **节点寻址**：7-bit 节点 ID（0-127），支持 127 个子节点
- **帧格式设计**：
  - 11-bit CAN ID = 4-bit 功能码 + 7-bit 节点 ID
  - 8-byte payload = 参数索引(1B) + 保留(3B) + 参数值(4B)
- **功能码分类**：
  - Alert (0x0)：从节点异常告警（最高优先级）
  - TimeSync (0x1)：时间同步广播
  - WriteSet (0x2)：主节点下发控制命令
  - Report (0x3)：从节点上报传感器数据

#### 2.2.3 分层软件架构
- **ESP32 主节点分层**：App → Service → ECUAL → MCAL → BSP
  - 关键服务：SensorState（多节点状态管理）、GUI Manager、VoiceAssistant、DeepSeekAPI
- **STM32 从节点分层**：firmware → crates/app → crates/bsw → crates/service
  - 关键模块：dispatcher（本地调度器）、PID 控制器、CAN 协议栈
- **层次间接口定义**：CAN 总线协议作为主从节点间唯一通信接口

### 2.3 控制策略设计（400-600 字）
#### 2.3.1 手动模式
- 主节点 GUI 选择目标从节点
- 触摸屏直接控制指定从节点的执行器
- 参数实时调整（风扇转速、灯光亮度）
- 支持多节点独立控制

#### 2.3.2 自动模式
- **本地自动调度**：每个 STM32 从节点独立运行迟滞调度器
- 迟滞控制策略（温度、湿度、土壤湿度、光照）
- 阈值配置：主节点下发，从节点本地执行
- 多节点并行控制，互不干扰

#### 2.3.3 AI 领航模式
- 语音交互流程：录音 → STT → LLM 推理 → TTS → 播放
- DeepSeek 大模型集成策略
- **多节点数据注入**：主节点聚合所有从节点传感器数据，注入 LLM 上下文
- **智能决策分发**：LLM 决策结果通过 CAN 总线下发到指定从节点执行

## 第3章 硬件电路设计（2,500-4,000 字）

### 3.1 系统硬件总体框图（300-500 字）
- 分布式多节点硬件架构图
- 主节点（ESP32-S3）硬件模块
- 从节点（STM32F407）硬件模块（×N）
- CAN 总线网络拓扑

### 3.2 主控芯片选型（400-600 字）
#### 3.2.1 ESP32-S3 选型分析
- 主频：240 MHz，双核 Xtensa LX7
- 内置 WiFi + 蓝牙 5.0
- 丰富的外设接口：SPI、I2C、I2S、UART
- 适用场景：网络通信、音频处理、GUI 驱动

#### 3.2.2 STM32F407VET6 选型分析
- 主频：168 MHz，ARM Cortex-M4F
- 内置 FPU，支持 DSP 指令
- 丰富的定时器资源：TIM1/TIM3（PWM、QEI）
- 适用场景：实时控制、传感器采集、电机驱动

### 3.3 传感器模块设计（600-1,000 字）
#### 3.3.1 SHT30 温湿度传感器
- 工作原理：I2C 接口，精度 ±0.3°C / ±2%RH
- 电路设计：上拉电阻配置、电源滤波
- 接口连接：PB6 (SCL)、PB7 (SDA)

#### 3.3.2 BH1750 光照传感器
- 工作原理：I2C 接口，量程 1-65535 Lux
- 电路设计：ADDR 引脚配置、去耦电容
- 接口连接：PB6 (SCL)、PB7 (SDA)

#### 3.3.3 土壤湿度传感器
- 工作原理：模拟量输出，ADC 采样
- 电路设计：分压电路、滤波电容
- 接口连接：PA0 (ADC1_IN0)

### 3.4 执行器模块设计（600-1,000 字）
#### 3.4.1 通风风扇驱动
- 电机类型：直流无刷风扇
- 驱动方式：PWM 调速（TIM1 CH1）
- 转速反馈：正交编码器（TIM3 QEI）
- PID 闭环控制电路

#### 3.4.2 水泵与加湿器驱动
- 驱动方式：继电器开关控制
- 电路设计：光耦隔离、续流二极管
- 接口连接：PA1 (水泵)、PA2 (加湿器)

#### 3.4.3 补光灯驱动
- 灯珠类型：WS2812B RGB 灯带
- 驱动方式：单总线协议（TIM2 CH2 DMA）
- 亮度与颜色可调

#### 3.4.4 遮阳舵机驱动
- 舵机型号：MG995
- 驱动方式：PWM 控制（TIM5 CH2, 50Hz）
- 角度范围：0°-300°（映射关系：0°=500μs, 300°=2500μs），实际使用 0°（收起）和 90°（展开）两个位置作为开关式遮阳执行器

### 3.5 通信模块设计（300-500 字）
#### 3.5.1 CAN 总线多节点网络设计
- 收发器芯片：TJA1051T（主节点 + 每个从节点各一片）
- 总线拓扑：总线型，主节点与所有从节点共享 CAN_H/CAN_L 总线
- 终端电阻：总线两端各 120Ω
- 波特率：1 Mbps（支持最大 127 个节点）
- 节点 ID 配置：从节点通过拨码开关或软件配置 7-bit 节点 ID（0-127）

#### 3.5.2 音频输入输出（主节点专用）
- 麦克风：INMP441（I2S 接口）
- 功放：MAX98357A（I2S 接口）
- 扬声器：8Ω 3W

### 3.6 电源模块设计（100-200 字）
- 本系统直接采用 STM32F407 与 ESP32-S3 开发板供电方案，未单独设计电源模块
- 开发板集成 5V → 3.3V LDO 稳压器，通过 USB 接口供电
- 传感器（SHT30、BH1750）由开发板 3.3V 引脚直接供电
- 执行器（风扇、水泵、舵机）由开发板 5V 引脚供电

### 4.1 软件架构概述（300-500 字）

### 4.3 PID 控制算法设计（800-1,200 字）
#### 4.3.1 离散位置式 PID 算法原理
- 数学公式推导：
  $$u(k) = K_p \cdot e(k) + K_i \cdot \sum_{j=0}^{k} e(j) \cdot \Delta t + K_d \cdot \frac{e(k) - e(k-1)}{\Delta t}$$
- 比例、积分、微分项的物理意义

#### 4.3.2 抗积分饱和策略
- 积分限幅（integral_limit）
- 输出限幅（output_limit）
- 条件积分分离

#### 4.3.3 通风风扇转速闭环控制
- 控制目标：设定转速 vs 编码器反馈转速
- PWM 输出：TIM1 CH1，20kHz 频率
- 编码器反馈：TIM3 QEI，4 倍频计数
- 默认参数：Kp=2.0, Ki=0.5, Kd=0.0
- 代码来源：`Src/Stm32_Control/crates/bsw/src/pid.rs`

#### 4.3.4 PID 参数远程调整
- CAN 总线接收 PID 参数更新
- 功能码：PidP (0x52)、PidI (0x53)、PidD (0x54)
- 参数热更新机制

### 4.4 CAN 协议栈设计（500-800 字）
#### 4.4.1 帧格式定义与多节点寻址
- 11-bit ID 结构：4-bit 功能码 + 7-bit 节点 ID
- 节点 ID 范围：0-127（0 为主节点，1-127 为从节点）
- 8-byte payload 格式：参数索引(1B) + 保留(3B) + 参数值(4B)
- 功能码分类：Report (0x3)、WriteSet (0x2)、TimeSync (0x1)、Alert (0x0)

#### 4.4.2 CAN 接收任务（can_rx）
- 帧解析逻辑：`can_proto_parse_packet()`
- **节点 ID 过滤**：从节点只处理目标节点 ID 匹配的帧（或广播帧）
- 数据分发：根据功能码更新本地 GLOBAL_STATE
- 错误处理：帧校验、保留位校验

#### 4.4.3 CAN 发送任务（can_tx）
- 帧组装逻辑：`can_proto_build_id()` + `can_proto_build_packet()`
- **节点 ID 填充**：发送帧携带本节点 ID
- 定时上报：传感器数据周期性发送（Report 功能码）
- 事件驱动：异常告警即时发送（Alert 功能码）

- 代码来源：`Src/Stm32_Control/crates/bsw/src/can_proto.rs`、`Src/Esp32_Interaction/src/Service/CanService/`

### 4.5 本地自动调度器设计（600-1,000 字）
#### 4.5.1 迟滞控制策略
- **本地闭环控制**：每个从节点独立运行调度器，无需主节点干预
- 温度控制：高温阈值 → 开风扇，低温阈值 → 关风扇
- 湿度控制：低湿度阈值 → 开加湿器，高湿度阈值 → 关加湿器
- 土壤湿度控制：干旱阈值 → 开水泵，湿润阈值 → 关水泵
- 光照控制：低光照阈值 → 开补光灯，高光照阈值 → 关补光灯

#### 4.5.2 调度器任务实现
- 任务周期：100ms
- 状态读取：从本地 GLOBAL_STATE 获取传感器数据
- 控制输出：更新本地执行器状态
- **阈值配置**：通过 CAN 接收主节点下发的阈值参数
- 代码来源：`Src/Stm32_Control/firmware/src/bin/domains/dispatcher.rs`

#### 4.5.3 控制模式切换
- 手动模式：调度器不干预，执行器由 CAN 命令直接控制（主节点远程操控）
- 自动模式：调度器根据阈值自动控制（本地闭环）
- 模式切换逻辑：通过 CAN 接收 ControlMode 命令（WriteSet 功能码）
- **分布式决策**：每个从节点独立维护自己的控制模式状态

### 4.6 全局状态管理（300-500 字）
#### 4.6.1 状态结构体设计
- `CurrentValues`：传感器读数（温度、湿度、光照、土壤湿度）
- `TargetValues`：执行器状态与设定值
- `ControlMode`：手动/自动模式枚举

#### 4.6.2 共享状态保护
- Embassy Mutex 互斥锁
- 任务间数据访问同步

## 第6章 总结与展望（800-1,500 字）

### 6.1 工作总结（300-500 字）
- 完成的主要工作
  - 双 MCU 架构设计与实现
  - CAN 总线多节点通信协议设计
  - 三种控制模式实现（手动、自动、AI 领航）
  - DeepSeek 大模型集成与语音交互
  - LVGL 图形界面设计（v9.3.0 + GUI Guider）

### 6.2 创新点（200-400 字）
- **分布式多节点架构设计**：ESP32 主节点 + 多个 STM32 从节点，通过 CAN 总线实现分布式控制，支持 127 个子节点扩展
- **大语言模型与温室控制的深度融合**：首次将 DeepSeek 大模型集成到分布式温室控制系统，实现"AI 领航"模式
- **多节点状态管理与数字孪生**：主节点维护所有从节点的实时状态，支持多区域协同分析与决策
- **本地闭环与远程调度结合**：STM32 从节点本地迟滞调度保证实时性，主节点远程调度实现全局优化
- **语音交互驱动的智能控制**：支持自然语言语音指令，聚合多节点数据进行智能问答

### 6.3 不足与展望（200-400 字）
#### 6.3.1 现有不足
- 传感器种类有限（缺乏 CO2、pH、EC 传感器）
- DeepSeek API 采用非流式返回（`stream: false`），需等待完整响应生成后才能输出，用户交互延迟较大
- 本地边缘 AI 能力不足，大模型推理完全依赖网络
- 趋势页与设置页尚未实现，历史数据查看与系统配置功能缺失
- 电源模块直接复用开发板 USB 供电，未设计独立电源管理电路

#### 6.3.2 未来展望
- 增加更多传感器类型（CO2、pH、EC）
- 将 DeepSeek API 改为流式返回（`stream: true`），提升用户交互体验
- 实现趋势页（多节点历史数据曲线对比）与设置页（节点管理、网络配置）
- 引入边缘 AI 模型（TinyML），减少对云端的依赖
- 设计独立电源管理模块，支持电池供电与低功耗模式
- 支持多温室集群管理
- 开发移动端 APP

### 附录 C CAN 协议功能码表

**CAN ID 结构**：11-bit = 4-bit 功能码 + 7-bit 节点 ID

| 功能码 | 值 | 方向 | 说明 | 优先级 |
|--------|------|------|------|--------|
| Alert | 0x0 | 从 → 主 | 异常告警 | 最高 |
| TimeSync | 0x1 | 主 → 从 | 时间同步广播 | 高 |
| WriteSet | 0x2 | 主 → 从 | 控制命令/参数配置 | 中 |
| Report | 0x3 | 从 → 主 | 传感器数据上报 | 低 |

**参数字典（节选）**

| 参数 | 索引 | 缩放因子 | 单位 |
|------|------|----------|------|
| LightMainPower | 0x10 | 1:1 | bool |
| WaterPump | 0x11 | 1:1 | bool |
| Humidifier | 0x12 | 1:1 | bool |
| VentilationFan | 0x13 | 1:1 | bool |
| SunshadeMotor | 0x14 | 1:1 | bool |
| Temperature | 0x30 | ×100 | °C |
| HumidityAir | 0x31 | ×100 | %RH |
| HumiditySoil | 0x32 | ×100 | %RH |
| LightIntensity | 0x33 | 1:1 | Lux |
| FanSpeed | 0x35 | 1:1 | RPM |
| PidP | 0x52 | ×100 | - |
| PidI | 0x53 | ×100 | - |
| PidD | 0x54 | ×100 | - |

**节点 ID 分配**
| 节点 ID | 角色 | 说明 |
|---------|------|------|
| 0 | 主节点（ESP32-S3） | 交互层，负责 GUI、语音、AI、状态管理 |
| 1-127 | 从节点（STM32F407） | 控制层，每个节点负责一个温室区域 |
