## 第1章 绪论（1,500-2,500 字）

### 1.1 研究背景与意义（500-800 字）
- 农业现代化与智能温室的发展趋势
- 物联网技术在农业环境监控中的应用
- 人工智能（大语言模型）在农业决策中的潜力
- 本课题的研究意义：构建"感知-决策-执行"一体化的智能温室系统

### 1.2 国内外研究现状（600-1,000 字）
- **国外研究现状**
  - 基于 PLC 的温室自动化控制系统
  - 基于 Raspberry Pi / Arduino 的开源温室监控方案
  - 基于云平台的远程温室管理（AWS IoT、Azure IoT）
- **国内研究现状**
  - 基于 STM32 的温室环境监测系统
  - 基于 ESP32 的 WiFi/蓝牙温室控制方案
  - 基于机器学习的温室环境预测与优化
- **现有研究的不足**
  - 缺乏大语言模型与温室控制的深度融合
  - 多为单 MCU 架构，扩展性与实时性难以兼顾
  - 用户交互方式单一（多为触摸屏或手机 APP）

## 第2章 需求分析与总体设计（2,000-3,000 字）

### 2.1 系统需求分析（600-1,000 字）
#### 2.1.1 功能需求
- 环境数据采集：温度、湿度、光照、土壤湿度
- 执行器控制：通风风扇、水泵、加湿器、补光灯、遮阳舵机
- 三种控制模式：手动模式、自动模式、AI 领航模式
- 用户交互：触摸屏 GUI、语音交互、实时数据显示
- 数据通信：CAN 总线、WiFi 联网、云端 API 调用

#### 2.1.2 性能需求
- 传感器采样周期：≤ 1s
- 控制响应时间：≤ 100ms
- CAN 总线通信速率：1 Mbps
- 系统稳定性：7×24 小时连续运行
- 语音识别准确率：≥ 90%

#### 2.1.3 非功能需求
- 低功耗设计
- 模块化与可扩展性
- 故障自检与容错机制

### 2.2 系统总体架构设计（600-1,000 字）
#### 2.2.1 双 MCU 架构设计
- ESP32-S3 交互层职责：GUI 显示、语音处理、WiFi 联网、AI 推理
- STM32F407 控制层职责：传感器采集、PID 控制、执行器驱动
- 架构优势分析：任务解耦、实时性保障、可扩展性

#### 2.2.2 分层软件架构
- ESP32 侧分层：App → Service → ECUAL → MCAL → BSP
- STM32 侧分层：firmware → crates/app → crates/bsw → crates/service
- 层次间接口定义

#### 2.2.3 通信架构设计
- CAN 总线拓扑：ESP32（主节点）↔ STM32（从节点）
- 帧格式定义：11-bit ID + 8-byte payload
- 功能码分类：Report (0x3)、Write/Set (0x2)、Alert (0x0)

### 2.3 控制策略设计（400-600 字）
#### 2.3.1 手动模式
- 触摸屏直接控制执行器
- 参数实时调整（风扇转速、灯光亮度）

#### 2.3.2 自动模式
- 迟滞控制策略（温度、湿度、土壤湿度、光照）
- 阈值配置与动态调整

#### 2.3.3 AI 领航模式
- 语音交互流程：录音 → STT → LLM 推理 → TTS → 播放
- DeepSeek 大模型集成策略
- 实时传感器数据注入

## 第3章 硬件电路设计（2,500-4,000 字）

### 3.1 系统硬件总体框图（300-500 字）
- 双 MCU 硬件架构图
- 主要功能模块划分

### 3.2 主控芯片选型（400-600 字）
#### 3.2.1 ESP32-S3 选型分析
- 主频：240 MHz，双核 Xtensa LX7
- 内置 WiFi + 蓝牙 5.0
- 丰富的外设接口：SPI、I2C、I2S、UART
- 适用场景：网络通信、音频处理、GUI 驱动

#### 3.2.2 STM32F407VET6 选型分析
- 主频：168 MHz，ARM Cortex-M4F
- 内置 FPU，支持 DSP 指令
- 丰富的定时器资源：TIM1/TIM3（PWM、QEI）
- 适用场景：实时控制、传感器采集、电机驱动

### 3.3 传感器模块设计（600-1,000 字）
#### 3.3.1 SHT30 温湿度传感器
- 工作原理：I2C 接口，精度 ±0.3°C / ±2%RH
- 电路设计：上拉电阻配置、电源滤波
- 接口连接：PB6 (SCL)、PB7 (SDA)

#### 3.3.2 BH1750 光照传感器
- 工作原理：I2C 接口，量程 1-65535 Lux
- 电路设计：ADDR 引脚配置、去耦电容
- 接口连接：PB6 (SCL)、PB7 (SDA)

#### 3.3.3 土壤湿度传感器
- 工作原理：模拟量输出，ADC 采样
- 电路设计：分压电路、滤波电容
- 接口连接：PA0 (ADC1_IN0)

### 3.4 执行器模块设计（600-1,000 字）
#### 3.4.1 通风风扇驱动
- 电机类型：直流无刷风扇
- 驱动方式：PWM 调速（TIM1 CH1）
- 转速反馈：正交编码器（TIM3 QEI）
- PID 闭环控制电路

#### 3.4.2 水泵与加湿器驱动
- 驱动方式：继电器开关控制
- 电路设计：光耦隔离、续流二极管
- 接口连接：PA1 (水泵)、PA2 (加湿器)

#### 3.4.3 补光灯驱动
- 灯珠类型：WS2812B RGB 灯带
- 驱动方式：单总线协议（TIM2 CH2 DMA）
- 亮度与颜色可调

#### 3.4.4 遮阳舵机驱动
- 舵机型号：MG995
- 驱动方式：PWM 控制（TIM4 CH1）
- 角度范围：0°-180°

### 3.5 通信模块设计（300-500 字）
#### 3.5.1 CAN 总线接口
- 收发器芯片：TJA1051T
- 终端电阻：120Ω
- 波特率：1 Mbps

#### 3.5.2 音频输入输出
- 麦克风：INMP441（I2S 接口）
- 功放：MAX98357A（I2S 接口）
- 扬声器：8Ω 3W

### 3.6 电源模块设计（200-400 字）
- 输入电压：12V DC
- 稳压方案：12V → 5V（LM2596）、5V → 3.3V（AMS1117）
- 电源分配：MCU、传感器、执行器独立供电

### 4.3 PID 控制算法设计（800-1,200 字）
#### 4.3.1 离散位置式 PID 算法原理
- 数学公式推导：
  $$u(k) = K_p \cdot e(k) + K_i \cdot \sum_{j=0}^{k} e(j) \cdot \Delta t + K_d \cdot \frac{e(k) - e(k-1)}{\Delta t}$$
- 比例、积分、微分项的物理意义

#### 4.3.2 抗积分饱和策略
- 积分限幅（integral_limit）
- 输出限幅（output_limit）
- 条件积分分离

#### 4.3.3 通风风扇转速闭环控制
- 控制目标：设定转速 vs 编码器反馈转速
- PWM 输出：TIM1 CH1，20kHz 频率
- 编码器反馈：TIM3 QEI，4 倍频计数
- 默认参数：Kp=2.0, Ki=0.5, Kd=0.0
- 代码来源：`Src/Stm32_Control/crates/bsw/src/pid.rs`

#### 4.3.4 PID 参数远程调整
- CAN 总线接收 PID 参数更新
- 功能码：PidP (0x52)、PidI (0x53)、PidD (0x54)
- 参数热更新机制

### 4.4 CAN 协议栈设计（500-800 字）
#### 4.4.1 帧格式定义
- 11-bit ID 结构：功能码 + 节点索引
- 8-byte payload 格式：根据功能码解析
- 功能码分类：Report (0x3)、Write (0x2)、Alert (0x0)

#### 4.4.2 CAN 接收任务（can_rx）
- 帧解析逻辑：`parse_rx_payload()`
- 数据分发：根据功能码更新 GLOBAL_STATE
- 错误处理：CRC 校验、超时检测

#### 4.4.3 CAN 发送任务（can_tx）
- 帧组装逻辑：`build_tx_frame()`
- 定时上报：传感器数据周期性发送
- 事件驱动：异常告警即时发送

- 代码来源：`Src/Stm32_Control/crates/bsw/src/can_proto.rs`

### 4.5 自动调度器设计（600-1,000 字）
#### 4.5.1 迟滞控制策略
- 温度控制：高温阈值 → 开风扇，低温阈值 → 关风扇
- 湿度控制：低湿度阈值 → 开加湿器，高湿度阈值 → 关加湿器
- 土壤湿度控制：干旱阈值 → 开水泵，湿润阈值 → 关水泵
- 光照控制：低光照阈值 → 开补光灯，高光照阈值 → 关补光灯

#### 4.5.2 调度器任务实现
- 任务周期：100ms
- 状态读取：从 GLOBAL_STATE 获取传感器数据
- 控制输出：更新执行器状态
- 代码来源：`Src/Stm32_Control/firmware/src/bin/domains/dispatcher.rs`

#### 4.5.3 控制模式切换
- 手动模式：调度器不干预，执行器由 CAN 命令直接控制
- 自动模式：调度器根据阈值自动控制
- 模式切换逻辑：通过 CAN 接收 ControlMode 命令

### 4.6 全局状态管理（300-500 字）
#### 4.6.1 状态结构体设计
- `CurrentValues`：传感器读数（温度、湿度、光照、土壤湿度）
- `TargetValues`：执行器状态与设定值
- `ControlMode`：手动/自动模式枚举

#### 4.6.2 共享状态保护
- Embassy Mutex 互斥锁
- 任务间数据访问同步

- 大模型推理延迟较高（网络依赖）
- 本地边缘 AI 能力不足

#### 7.3.2 未来展望
- 增加更多传感器类型
- 引入边缘 AI 模型（TinyML）
- 支持多温室集群管理
- 开发移动端 APP
- 集成气象数据预测
