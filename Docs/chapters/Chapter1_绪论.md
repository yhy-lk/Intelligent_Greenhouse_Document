# 第1章 绪论

## 1.1 研究背景与意义

农业是国民经济的基础产业，温室设施农业通过人为创造适宜作物生长的环境条件，突破了自然气候对农业生产的季节性限制。随着全球人口增长与耕地缩减的矛盾日益加剧，提高单位面积产量和资源利用效率已成为农业可持续发展的核心命题[@karunathilake2023smart]。然而，传统温室系统的控制逻辑停留在基于阈值的开关控制层面，缺乏对复杂环境变化的自适应能力[@shamshiri2018advances]。近年来，以大语言模型（Large Language Model, LLM）为代表的人工智能技术取得了进展，其自然语言理解与生成能力为农业智能决策开辟了新的技术路径[@tzachor2023llm]。

基于上述分析，本课题设计并实现了一种基于 ESP32 与 STM32 的分布式智能温室控制系统。系统采用"ESP32 主节点 + 多个 STM32 从节点"的分布式架构，主节点负责交互层功能，从节点负责控制层功能，主从节点之间通过 CAN 总线实现可靠通信。系统集成了 DeepSeek 大语言模型，实现了手动控制、自动控制和 AI 领航三种控制模式，构建了"多节点感知、集中决策、分布式执行"的智能温室控制体系。

## 1.2 国内外研究现状

### 1.2.1 国外研究现状

在智能温室领域，国外研究起步较早。早期系统基于 PLC 构建，可靠性高但成本高昂、扩展困难[@shamshiri2013review]。随着嵌入式系统和开源硬件的普及，基于 Raspberry Pi 和 Arduino 等平台的低成本温室监控方案逐渐兴起[@mahbub2020smart; @soheli2022smart]。近年来，云计算平台的引入进一步拓展了温室管理的远程化与智能化水平[@morchid2024irrigation; @hooshmand2025smart]。在人工智能与农业的交叉研究方面，Tzachor 等[@tzachor2023llm]和 Shaikh 等[@shaikh2025llm]分别探讨了大语言模型在农业知识问答和决策支持中的应用潜力。然而，将大语言模型与嵌入式温室控制系统深度集成的研究尚处于探索阶段。

### 1.2.2 国内研究现状

国内在智能温室领域同样取得了进展。在 STM32 平台方面，Wei[@wei2022intelligent]、Ma 等[@ma2025stm32monitor]、Li 和 Wu[@li2025rice]以及 Hong[@hong2020tobacco]分别将 STM32 应用于温室温控、环境监测、水稻育秧和烟草种植等场景。在 ESP32 平台方面，Zhao 等[@zhao2023esp32greenhouse]和 Bustomi 等[@bustomi2025esp32]利用 ESP32 的 WiFi 功能实现了温室数据的无线传输与远程监控。在控制算法方面，Huang 等[@huang2024vegetable]和 Zeng 等[@zeng2012nonlinear]分别采用单神经元 PID 和 RBF 神经网络提高了控制系统的自适应能力。

### 1.2.3 现有研究的不足

综合分析国内外研究现状，现有智能温室系统仍存在以下不足：

（1）**大语言模型与温室控制的融合不足**。现有系统主要依赖预设规则或传统控制算法，缺乏基于大语言模型的智能决策能力[@tzachor2023llm]。

（2）**单 MCU 架构为主，扩展性与实时性难以兼顾**。单一控制器在规模扩大时处理能力成为瓶颈，且单点故障会导致系统瘫痪[@wei2022intelligent]。

（3）**用户交互方式单一**。大多数系统局限于触摸屏或手机 APP，缺乏语音交互等更自然的人机交互方式[@bustomi2025esp32]。

（4）**分布式架构的工程实践不足**。将 CAN 总线应用于温室多节点分布式控制、并结合大语言模型实现闭环控制的完整方案尚不多见[@sumalan2020greenhouse]。

## 1.3 研究内容与技术路线

针对上述不足，本课题围绕"分布式多节点架构 + 大语言模型智能决策"的核心理念，开展以下研究工作：

（1）**分布式多节点架构设计**。设计"ESP32 主节点 + 多个 STM32 从节点"的分布式系统架构，实现任务解耦与功能分离。

（2）**CAN 总线多节点通信协议设计**。基于 CAN 2.0A 标准设计适用于温室场景的多节点通信协议，支持最多 127 个从节点的寻址与通信。

（3）**三种控制模式实现**。设计并实现手动控制、自动控制和 AI 领航三种控制模式，其中 AI 领航模式集成 DeepSeek 大语言模型，实现语音交互驱动的智能环境调控。

技术路线方面，本课题遵循"需求分析、硬件设计、软件设计、系统集成、测试验证"的工程开发流程。

## 1.4 论文组织结构

本论文共分为六章：第1章绪论，阐述研究背景、现状与不足，明确研究内容；第2章进行需求分析与总体架构设计；第3章介绍硬件电路设计；第4章详述 STM32 控制层软件设计；第5章详述 ESP32 交互层软件设计；第6章总结全文并展望未来工作。
