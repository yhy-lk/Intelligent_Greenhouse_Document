# 第一章 绪论

## 1.1 研究背景与意义

农业作为国民经济的基础产业，不仅关乎国家的粮食安全与社会稳定，更在日益严峻的资源环境约束下面临着向集约化、精细化与智能化方向转型的迫切需求。传统农业露天生产模式严重依赖自然气候条件，抵抗极端天气、病虫害等系统性风险的能力极其有限。在此背景下，设施农业（Facility Agriculture）作为现代农业技术的重要标志应运而生。通过在相对封闭的环境空间中人为构建适宜作物生长的微气候条件，温室控制突破了土地资源与季节因素的瓶颈限制，极大地提高了单位面积的经济效益与资源利用率[@karunathilake2023smart; @bagagiolo2022greenhouse]。然而，随着温室规模的不断扩大和作物对环境精度要求的日益提高，对温室内部诸如温度、湿度、光照强度等环境要素进行精准、高效调控的技术难度也呈几何级数增长。温室微气候实质上是一个具有大时滞性、强耦合性与非线性的复杂多变量动态系统[@shamshiri2018advances; @su2020pid]。传统的温室控制器大多采用基于静态阈值的偏差开关控制或简单的比例调节，这种刚性且粗放的控制策略不仅容易导致设备频繁启停带来机械损耗，更难以应对动态变化的气候扰动，常常出现“越界震荡”和资源无效浪费现象[@hu2014automatic; @subahi2020iot]。

近年来，伴随物联网（Internet of Things, IoT）、边缘计算（Edge Computing）与分布式系统技术的高速演进，温室控制逐渐由集中式结构向分布式节点协同控制演化[@navarro2020iot; @sumalan2020greenhouse]。其中，基于单片机或微控制器（MCU）构建分布式边缘检测与控制节点已成为行业实践的主流架构[@wei2022intelligent; @ma2025stm32monitor]。然而，在实际工程应用中，单点 MCU 既要处理日益繁重的传感器数据采集与硬件底层闭环控制任务，又要兼顾上层网络通信及复杂的用户图形界面交互，往往面临严重的算力瓶颈与架构臃肿问题，甚至会因为单节点故障导致全系统瘫痪，难以在实时性与可扩展性之间取得平衡[@liu2015intelligent; @bustomi2025esp32]。此外，虽然以大语言模型（Large Language Model, LLM）为代表的通用人工智能技术在农业领域的知识问答和辅助决策方面初步展示出颠覆性潜力[@tzachor2023llm; @shaikh2025llm]，但这些模型尚未深入到温室系统底层的设备执行链路，大多停留在“离线建议”阶段。现有的温室控制系统在人机交互层面几乎完全依赖预置规则构成的触摸面板或手机 APP 界面，缺乏适应农民群体尤其是老年人群体操作特点的自然语言交互手段，限制了系统的普适性与智能化程度。

针对上述由集中控制向分布式演进的技术阻碍，以及物联网设备与通用人工智能融合度不足的现实痛点，本课题致力于研究并提出一种结合分布式硬件架构与云端大语言模型智能决策的温室控制新范式。研究旨在通过底层计算节点与顶层交互网络节点的工程解耦，突破单微控制器的性能束缚；同时，在系统决策链条中深度引入自然语义分析模型，打破“人-机”之间传统的指令硬编码壁垒。

研发此类系统面临三项核心挑战：首先，如何在主从分布式异构处理器（如 Wi-Fi SoC 与实时 MCU）之间构建一个兼顾高吞吐量、低延迟和强抗干扰能力的局域通信网络，并建立统一的设备挂载与心跳保活机制；其次，怎样克服云端 LLM 存在的不定回复时延及大段非结构化自然语言带来的解析难题，将其平滑转换为本地执行器可识别的确定性控制参数簇；最后，在底层驱动端，面对高低压设备交叉并存、传感反馈具有明显惯性迟滞的恶劣电气环境，如何设计具备强抗饱和能力的高频离散闭环算法（如增量式 PID 等）以抑制外部扰动影响。

为此，本课题设计并实现了一套以 ESP32 交互型微控制器协同多台 STM32 执行型微控制器的分布式智能温室系统。方案借助车规级 CAN（Controller Area Network）总线通信机制桥接跨芯片底层通信，利用其硬件仲裁与多主广播特性彻底解除各模块的时序强耦合[@niveditha2018can]；软件架构层面，ESP32 充当“交互与决策中枢”，对接 DeepSeek 云端大语言模型接口，设计基于多级正则匹配与 JSON 强制约束的提示词调度词法，将用户的模糊语境精准映射至设备硬件动作序列；下游 STM32 阵列则专职“稳健执行”，依托内部硬件定时器构筑了包含编码器反馈、时钟限幅保护机制的高频微秒级 PID 控制循环（闭环周期精度控制在 10ms 量级），从而彻底实现了环境闭环自适应与算力解耦任务剥离。

本研究的贡献主要体现在以下四个方面：第一，在系统架构（硬件电路与总体需求）方面，结合了 STM32 与 ESP32 的双架构优势，克服了单机系统可拓展性差与高负载迟滞的痼疾（见第二、三章）；第二，在底层控制策略上，提出并落地了基于编码器负反馈的高频数字 PID 参数自整定循环方案，确保了调控过程的超调极小化（见第四章）；第三，通信协议创新方面，在纯裸机环境中完整抽象了基于 CAN 2.0A 的软总线寻址、拆包与重传协议模型，大幅降低了总线碰撞率（见第四章）；第四，在人机交互与 AI 整合维度，创新性引入了以智能大模型充当意图领航系统（AI 领航模式）的工作流，填补了现有嵌入式农业设备在此领域的空白（见第五章）。本论文不仅在温室精细环境管理的嵌入式实现中提出了有效解决方案，同时为后续类似基于多节点传感及人工智能调度架构的工业现场设计提供了具有参考价值的工程范本。
## 1.2 国内外研究现状

### 1.2.1 国外研究现状

设施农业及其环境监控技术在欧美等农业发达国家起步较早，其演进历程呈现出从底层重型工业控制向轻量化、网络化、智能化逐步升级的脉络。在 20 世纪末至 21 世纪初，国外的智能温室系统多建立在可编程逻辑控制器（PLC）和大型工业测控网络（如 SCADA 系统）的基础之上[@shamshiri2013review]。这类系统虽然具备极高的工业级可靠性，并且在控制执行逻辑、长期无故障运行方面非常成熟，但高昂的硬件成本、封闭的专有通信协议以及扩展部署时的施工复杂度，使得其仅适用于大型科研机构或具备大量资金支持的集约化农业工厂。传统的基于规则的开关控制机制虽然稳定性高，却无法灵活应对非线性的气候变化扰动，难以在广大的中小型农户群体中得到快速普及。

随着无线传感器网络（WSN）、物联网核心技术（IoT）以及微机电系统（MEMS）的迅猛发展，国外的研究重点迅速转向了以低成本、低功耗嵌入式平台驱动的新型云边协同测控系统。Mahbub 等[@mahbub2020smart]系统地提出了基于智能嵌入式电子设备的智慧农业体系概念构架，论述了低功耗网络传感器阵列在农业大面积多节点动态监测中的可行性。Subahi 等[@subahi2020iot]通过引入定制的网络中间件与无线通信模块，设计了一套可以实时采集温度与控制反馈的物联网监测评估系统，有效降低了传感器节点的数据传输功耗。Morchid 等[@morchid2024irrigation]则结合了遥测数据采集与云计算边缘服务器，设计了一款着眼于增强农业用水安全与水资源精确调度管理的智能灌溉系统。上述研究标志着国外在温室物联网基础建设层面已进入“云-边-端”三层架构的深度普及期，极大缓解了本地计算资源的压力。

进一步在系统智能化层面上，机器学习（ML）与数据流挖掘范式开始逐渐走入温室气候模型的预测之中。例如，Soheli 等[@soheli2022smart]不仅评估了各类温室传感器读数的协同联动综合表现，还创新性地引入了传统的监督学习与分类模型，用于温室农作物的病害特征识别和中长期生长趋势预测。在自然语言处理（NLP）及生成式大模型前沿应用领域的探讨上，Tzachor 等人[@tzachor2023llm]详细调研并前瞻性地指出，将超级大规模语言模型（LLM）嵌入农业知识及推广应用服务网络中，能够极大地重塑农业技术推广咨询服务的形态。随后，Shaikh 等[@shaikh2025llm]更是深入剖析了 LLM 在充当农用作物诊断与咨询系统的智能潜力，认为凭借超强的上下文长程记忆、逻辑推理及多语言理解能力，LLM 大幅拉近了顶尖农学技术指导与普通一线劳作者的沟通距离。然而，综合这些最前沿文献可以发现，国外此类先进的 AI 模型探索仍主要局限于顶层软件咨询层、决策辅助建议看板以及视觉病理特征检索。真正以边缘计算微控制器为下沉载体、直接将大语言模型生成的非结构化决策推理穿透到底层硬件执行器进行实时物理环境参数闭环调节的研究实践，依然受制于现网延时、算子边缘适配及硬件驱动安全等重重工程验证空白。

### 1.2.2 国内研究现状

近年来，随着国家一系列“乡村振兴”与“智慧农业”扶持政策的出台，国内在温室大棚精准测控领域实现了从测控理论摸索向大规模落地工程的飞跃。相较于国外往往依赖大型商业化集成方案或昂贵云服终端设备，国内学术界与工程界尤为侧重利用成本低廉、开源社区生态极为完善的主流 MCU（如基于 ARM Cortex-M 内核的 STM32 系列单片机及乐鑫 ESP 射频系列芯片）来实现高度定制化的环境监测与执行控制板。

在温室底层控制节点终端研究与专业化应用适配方面，Wei[@wei2022intelligent] 详细讨论了一种由 STM32 主导的智能温室温度控制技术实现，通过集成数字温湿度传感器收集现场物理参量并完成轻量级的阈值限位保护动作。Li 和 Wu[@li2025rice] 与 Hong[@hong2020tobacco] 则进一步发掘了微控制器在复杂特种农业上的应用潜力，他们分别专门针对水稻催芽育秧温室的阶段性控温、以及烟草醇化专用烘烤房的严苛热力学温湿度工艺要求，提出了采用 STM32 芯片全权执行环境多点采样流水线与加热、冷却、排湿联动设备群的成套工程方案。在注重宏观管理层与网络接入节点组合的研究中，Ma 等[@ma2025stm32monitor] 以及 Chen[@chen2024optimization] 着重探讨了单片机通过扩展模块接入 OneNET 或者阿里云（Aliyun）等商用物联网中台用于后台大屏数据统筹分析、预警推送与海量监测数据持久化储存的具体技术联接方式。

另一方面，作为在无线网络互联架构与高集成度片上开发方面成绩斐然的国产 SoC——ESP32 系列芯片，近年来受到了国内物联网科研工作者的强烈青睐。Zhao 等人[@zhao2023esp32greenhouse] 提出了一种极具工程应用示范意义的基于 ESP32 的轻量级智能温室控制台系统方案；在此网络化硬件基础上，不仅国内的研究群体，包括海内外众多的开源应用开发者（如 Bustomi等[@bustomi2025esp32]、Hercog等[@hercog2023esp32]）也不断挖掘 ESP32 内置实时操作系统（如 FreeRTOS）的多任务并发能力和自带 TCP/IP 协议栈硬件卸载功能，进行实时温室传感数据桥接和本地轻设备网页伺服渲染。然而，功能堆叠往往带来瓶颈，这些单节点全包揽式的设计愈发面临中断响应不及时的问题。

此外，由于温室内部环境是一个典型的具备强耦合性、大滞后性与时变性的非线性物理系统（例如开启天窗遮阳网的同时会不可避免地导致温室急剧降温且气流受阻），国内众多学者在温室底层调节算法模型优化的跟进研究同样具备深远的指导价值。传统固定参数数字 PID（比例-积分-微分）常常因无法动态响应突然侵入的巨大温差梯度而容易陷入限幅“失控”或者剧烈震荡。为此，Su 等人[@su2020pid]和 Huang 等人[@huang2024vegetable]在算法改进上发力，分别提出了基于在线参数自整定期望方程、以及基于单神经元网络的自适应改进版 PID 以应对大时滞滞后特性；Zeng 等[@zeng2012nonlinear]则更早地引入了面向强非线性特性的非线性降阶算法。这些算法学理论探索对本系统实现基于硬件定时器与编码器底层负反馈调节防震荡执行逻辑，有着极强的技术启示。

### 1.2.3 现有研究的不足

综合上述针对国内外智能温室大棚在电气自动化、传感物联网化与决策智能化演进轨迹的全面细致梳理，尽管现行大量通用技术已经在数据无线云端透传、环境指标阈值限位与单点控制器机构等局部子领域趋于成熟完善，但若以“全场景无缝感知、全局边缘自适应协同与自然语义极简人机互通”这一新时代农业 AI 需求为审视标尺来看，目前的系统架构与实施方案仍旧暴露出以下明显的工程局限与研究断层：

（1）**大语言模型与边缘重度测控执行链路的业务融合依然停留在概念外围**。纵观当前国内外前沿尝试接入 LLM 的农业辅助验证案例，绝大多数仍然止步于利用高并发集中云服务体系构建的单纯信息分发型“对话机器人”。未能实质性深入探索并打通大语言模型对系统最底层执行逻辑（如水泵的转速、补风扇的 PWM 占空比等硬核微控指令）的解构、约束与直接下放转换[@tzachor2023llm]。学术界长期缺乏将非确定化自然语言模糊意图精准映射、安全落地至具备严格时序要求的工业级微小电气控制层面的完整研究实践支撑。

（2）**以单核心微控制器为主的应用架构日益呈现严重的算力瓶颈与系统脆弱性**。目前可见的开源方案和许多研究设计，绝大部分仍然采用一块单片机（诸如 STM32 或 ESP8266/ESP32）包揽整个温室设备生命周期任务的粗放做法。这些任务同时掺杂了外部传感器的 I2C 高频轮询通信、大功率继电器阵列的强电硬控、庞大复杂的本地 LCD 物理屏幕画面渲染更新逻辑，以及面临着由于信号波动可能引发长时间阻塞死锁的 TCP/IP 接入等[@wei2022intelligent]。此种重度耦合模式容易导致主控时序错乱，单点假死频发；更暴露出横向拓展性极差（Scalability）的痛点，在增设更大跨度农田面积传感器阵列时显得捉襟见肘，难以兼顾外设实时响应性与交互流畅度。

（3）**用户物理交互方式陷入固态程式化和孤岛化桎梏**。大多数现行落地的智控管理系统，其人机交互入口不仅高度依赖物理实体界面的触摸输入（HMI 屏幕、薄膜按键面板），或者是需要深度绑定各类强管控的商业平台与微信小程序网络环境[@bustomi2025esp32]。这种封闭的体验使得用户操作充满了开发人员前置硬编码的程式门槛。缺少既能兼顾离线安全容错，又能支持诸如语音非接触解析输入或是大尺度自然口语化聊天式引导的操作入口体系，严重限制了新平台对相对排斥繁琐界面学习的老龄非专业农业群体用户的可用化扩散能力。

（4）**成熟工业级分布式通信架构在低成本温室局域网系统中的微缩部署实践严重匮乏**。在广袤巨型的现代化温室长廊设施之中，虽然多见 PLC 配合 RS485 差分有源结构的串联拉通布局；但面对经费极端敏感、监测节点地理分布呈现高度碎片散点状、并且同时并存微瓦级数字传感器与巨瓦级水泵风机干扰源的中小型智慧大棚领域，市面上少有能够系统且完整地向前端微控制器下放汽车工业级现场 CAN 总线（Controller Area Network）多节点挂载优势的廉价方案模型。现有一部分企图运用普通 I2C 总线延长线或是 USART 点对点通讯强行拼凑的分离式设计，一旦遭遇强电磁脉冲或瞬态启停干扰，极易引发时序崩塌与协议僵死，从根本上破坏了边缘分布式主从架构赖以正常协同运转的物理通信基石[@sumalan2020greenhouse]。

## 1.3 研究内容与技术路线

针对现有单控制核心温室监测结构存在的高强耦合瓶颈、恶劣环境下的总线通信易扰动以及智能决策与指令执行机制高度割裂的不足，本课题严格围绕“基于 CAN 总线的底层分布式多节点网络解耦架构”结合“大语言模型赋能的顶层模糊意图识别”的核心创新视角，展开系统性突破方案研究。课题具体研究内容主要划分为三大维度：

（1）**分布式异构双重控制架构的硬件构建与解耦设计**。突破将交互展示网络传输与底层时序控制合并于单芯片处理的局限框架，提出并设计“ESP32 交互主节点协同多个 STM32 执行从节点”的分布式部署系统硬件与资源调度方案体系。确保上层主节点仅承担多线程环境下的网络通信、语音收集发送及 GUI 界面渲染工作；而各分区并行的 STM32 专属从节点只聚力于当地子模块环境区域内（如 SHT30、BH1750 通信）的硬件控制链路维护及基于内部硬件定时资源的高频电控（采用包含编码器反馈闭环与 PWM 驱动机制的水泵、风机控制），实现任务隔离与电气干扰隔离的双重分离。

（2）**面向温室场景的工业级多控制子节点 CAN 软总线协议设计**。立足分布式节点的物理互通需求，放弃长距离抗干扰弱的电平直驱或者简单的 USART 全双工跨线传输机制；基于被汽车行业广泛验证的 CAN 2.0A 协议收发硬件，面向由于温室多点位测控所需设计的自定义应用层封包格式机制结构构建了多节点防冲突软挂载方案。协议涵盖报文格式自定、地址掩码分配寻址以及基于标准帧标识符（ID）的心跳检测与错误丢包重传策略，确保上层长指令拆分后的顺序拼接严密，从而建立至多可挂载 127 处分散监测域的可靠信息传输系统基石。

（3）**基于 DeepSeek LLM 多模态解析机制及三种环境智控模式落地设计**。在完成双微控制器跨通讯总线信息自由流转的基础上，重点攻克在软件抽象逻辑层面制定兼容“人工经验干预—硬规则固态调度—AI大模型决策调频”全链条的三级控制权移交系统架构。深入编码并联调验证手动控制、自动控制以及全新的最具特征性的“AI 领航模式”。设计多层 API 请求中继处理服务器中间件环境，不仅处理云侧大语言模型基于提示词（Prompt）严格限定结构体回传输出数据的网络解包强转类型难题，同时完成利用自然语音进行意图采集的底层驱动调用。构建真正的具备“多节点分布式感知、汇聚云端集中处理决策、向下放权分布式驱动执行”的物联网温室全智能化运作范本。

在研究技术路线的设计与工程管理推进执行方面，本课题自底向上遵循瀑布式迭代验证法开发生命周期。首先，实施精确的需求分析与理论概念选型，确定具体物理技术与指标限制范围（如 24V/12V 下降驱动、温差范围约束定级）；接着，着手外围关键硬件基础电路原理图方案搭建与传感器接口确认；在确定电路验证具备电气完整性后，分为两条代码战线平行研发——一方面使用基于 STM32 生态的数字 PID 及 CAN 外设基础裸机控制程序，另一方面深耕基于 ESP32 的 HTTP 网络接口、云端授权代理及嵌入式界面图形库（LVGL）绘制；最终，通过总线贯通联合打通双向数据流通道，将所有单体软件组件融合成闭环系统并部署严苛模拟温室多场景边界压力环境中的综合抗干扰功能与性能验收测试。

## 1.4 论文组织结构

为逻辑清晰、主次分明地完整阐述系统的研发链路，本论文内容的组织结构共划分为六章进行编排详述：

**第一章 绪论**：全方位阐述本项研究的宏观农业背景基础、科学探究意义，深挖并详尽评析当前国外与国内对于嵌入式控制设备运用的学术与工程现状及其瓶颈暴露点，顺势明确本论文的研究目标、具体研究剖析层面与既定技术线路。

**第二章 需求分析与总体设计**：从用户农业操控人员的刚性需求出发，提出定量的功能技术与非功能属性指标拆解；绘制勾勒贯穿总体设计蓝图和通讯骨架，构建从 ESP32 发出调度指令横跨 CAN 网络及至底层受控节点的系统体系雏形。

**第三章 硬件电路设计**：详明本系统中两大异构核心的主频、外设硬件选型理念；逐模块剖析电机驱动、光耦隔离继电器触发链路以及高精数字传感器接口电路设计的电气考量，奠定坚实的系统物理运行基础。

**第四章 STM32 控制层软件设计**：详细解构属于核心底层动作执行区的固件程序结构，着重于针对大惯性物理参量的 PID 闭环运动控制核心代码实现、轮询式时序状态机流转框架，以及自定义 CAN 2.0A 报文与应用层指令重组组装的核心解析策略实现思路分析。

**第五章 ESP32 交互层软件设计**：集中剖析上位界面处理方案架构，涵盖图形界面库布局构建、云端 DeepSeek 模型调用请求的 JSON 序列化解压包装处理机制逻辑、语音识别交互唤醒过程控制以及跨平台数据中转统领调配过程。

**第六章 总结与展望**：系统性全链路概括本课题实现的功能闭环设计特色及创新突破进展点，同时本着实事求是的科学态度分析目前受限时间与算力条件可能存在的短板，并在此基础之上提出针对后延研究领域的下一步演进展望。
