# 第一章 绪论

## 1.1 研究背景与意义

### 1.1.1 农业现代化与智能温室的发展需求

农业作为国民经济的基础产业，其发展水平直接关系到国家粮食安全和社会稳定。随着全球人口的持续增长和城市化进程的加速推进，传统农业生产模式面临着前所未有的挑战。据联合国粮农组织（FAO）预测，到 2050 年全球人口将达到 97 亿，粮食需求将比当前增加约 70%[@morchid2024irrigation]。在此背景下，如何提高农业生产效率、保障农产品质量、实现农业可持续发展已成为全球性课题。

温室农业作为现代设施农业的重要组成部分，通过人工构建可控的生长环境，能够有效突破自然气候条件的限制，实现农作物的周年生产和高产稳产。与传统露天种植相比，温室栽培具有显著优势：首先，温室环境能够有效隔离外界不利气候因素，如暴雨、冰雹、极端温度等，为作物提供稳定的生长条件；其次，通过精确调控温度、湿度、光照、二氧化碳浓度等环境参数，可以显著提高作物产量和品质；最后，温室农业能够有效减少农药和化肥的使用量，降低对环境的污染[@abou2025review]。

然而，传统温室管理主要依赖人工经验，存在劳动强度大、管理粗放、资源浪费严重等问题。随着物联网（Internet of Things, IoT）技术、嵌入式系统和人工智能技术的快速发展，智能温室系统应运而生。智能温室通过部署各类传感器实时采集环境参数，利用微控制器进行数据处理和决策，并通过执行机构自动调节环境条件，实现了温室管理的自动化、精准化和智能化[@liu2015intelligent]。

### 1.1.2 物联网技术在农业领域的应用现状

物联网技术是智能温室系统的核心支撑技术之一。物联网通过将各种信息传感设备与互联网结合起来，实现物与物、物与人的泛在连接，从而实现对物品的智能化识别、定位、跟踪、监控和管理。在农业领域，物联网技术的应用已经渗透到农业生产的各个环节，包括环境监测、精准灌溉、智能施肥、病虫害防治等[@mahbub2020smart]。

在温室环境监测方面，物联网系统通常由感知层、传输层和应用层三个层次组成。感知层负责采集温室内的温度、湿度、光照强度、土壤水分、二氧化碳浓度等环境参数；传输层负责将采集到的数据传输到数据处理中心；应用层则负责数据的存储、分析和可视化展示，并提供远程控制功能[@soheli2022smart]。

近年来，基于物联网的智能温室系统在国内外得到了广泛应用。在国外，荷兰、以色列、日本等农业发达国家已经建立了较为成熟的智能温室管理体系，实现了温室环境的精确控制和资源的高效利用。在国内，随着国家对农业现代化的重视和物联网技术的普及，智能温室系统也得到了快速发展，广泛应用于蔬菜种植、花卉培育、果树栽培等领域[@subahi2020iot]。

### 1.1.3 嵌入式系统在智能温室中的关键作用

嵌入式系统是智能温室系统的"大脑"，负责数据的采集、处理和控制指令的生成。与通用计算机系统相比，嵌入式系统具有体积小、功耗低、成本适中、可靠性高等优点，非常适合应用于温室环境监测和控制场景。在智能温室系统中，嵌入式系统通常承担以下核心任务：

第一，传感器数据采集与处理。嵌入式系统通过 I2C、SPI、ADC 等接口与各类传感器连接，实时采集温室环境参数。例如，SHT30 温湿度传感器通过 I2C 总线提供高精度的温湿度数据，BH1750 光照传感器用于测量光照强度，土壤湿度传感器用于监测土壤含水量。嵌入式系统需要对这些原始数据进行滤波、校准和单位转换等预处理操作[@sht30datasheet; @bh1750datasheet]。

第二，控制算法执行。根据采集到的环境参数和预设的目标值，嵌入式系统需要执行相应的控制算法来调节执行机构。常见的控制算法包括开关控制（迟滞控制）、PID 控制、模糊控制等。其中，PID 控制算法因其结构简单、参数整定方便、鲁棒性好等优点，在温室环境控制中得到了广泛应用[@ang2005pid]。

第三，通信与数据交换。智能温室系统通常采用分布式架构，由多个嵌入式节点组成。这些节点之间需要通过可靠的通信协议进行数据交换和协同工作。CAN（Controller Area Network）总线作为一种成熟的现场总线技术，具有高可靠性、实时性和灵活性等优点，被广泛应用于汽车电子、工业控制等领域，近年来也逐渐在农业物联网系统中得到应用[@zhao2023esp32greenhouse]。

第四，人机交互与显示。为了方便用户监控温室状态和设置控制参数，嵌入式系统通常配备有图形用户界面（GUI）。LVGL（Light and Versatile Graphics Library）是一个开源的嵌入式 GUI 库，支持多种显示控制器和输入设备，能够提供丰富的界面元素和流畅的用户体验，非常适合应用于资源受限的嵌入式平台。

### 1.1.4 本课题研究的意义

基于上述背景，本课题设计并实现了一种基于 ESP32 和 STM32 双处理器架构的智能温室系统。该系统采用分层设计理念，将人机交互、网络通信等复杂功能分配给 ESP32 处理器，将实时性要求高的传感器数据采集和控制算法执行分配给 STM32 处理器，通过 CAN 总线实现两个处理器之间的数据交换和协同工作。

本课题的研究具有以下重要意义：

首先，从技术层面来看，本系统采用的双处理器架构有效解决了传统单处理器系统在处理能力和实时性方面的瓶颈问题。ESP32 处理器具有强大的网络连接能力，支持 WiFi 和蓝牙通信，能够方便地接入互联网，实现远程监控和数据上传。STM32 处理器具有优秀的实时处理能力，能够精确控制采样周期和控制周期，确保控制系统的稳定性和响应速度。这种分工协作的架构设计，既发挥了 ESP32 在网络通信和人机交互方面的优势，又充分利用了 STM32 在实时控制方面的特长，实现了系统性能的最优化[@zhu2020embedded]。

其次，从应用层面来看，本系统集成了多种环境传感器和执行机构，能够实现对温室温度、湿度、光照、土壤水分等关键环境参数的全面监测和自动控制。系统支持手动控制、自动控制和 AI 智能控制三种工作模式，用户可以根据实际需求灵活选择。特别是 AI 智能控制模式，通过集成 DeepSeek 大语言模型 API，实现了基于自然语言交互的智能温室管理，为农业智能化提供了新的解决方案。

最后，从学术层面来看，本课题的研究涉及嵌入式系统设计、传感器技术、控制理论、通信协议、人机交互等多个学科领域，具有较强的综合性和实践性。通过对智能温室系统的设计与实现，可以深入理解物联网系统的架构设计、嵌入式软件开发、实时控制算法等关键技术，为今后从事相关领域的研究和开发工作奠定坚实的基础。

综上所述，本课题的研究对于推动农业现代化、促进物联网技术在农业领域的应用、培养具有工程实践能力的高级技术人才具有重要的理论意义和实践价值。

## 1.2 国内外研究现状

### 1.2.1 国外研究现状

国外智能温室技术的研究起步较早，经过数十年的发展，已经形成了较为成熟的技术体系和产业规模。荷兰、以色列、日本、美国等发达国家在智能温室领域的研究处于世界领先水平，其技术特点和应用模式各有侧重。

荷兰作为全球温室技术最先进的国家之一，其温室面积虽然仅占全国农业用地的 0.5%，但产值却占农业总产值的 35%以上。荷兰温室以玻璃温室为主，采用先进的环境控制系统，能够精确调节温度、湿度、光照、二氧化碳浓度等参数。荷兰瓦赫宁根大学（Wageningen University & Research）在温室环境建模、控制算法优化、能源管理等方面开展了大量研究，其开发的温室气候控制模型被广泛应用于世界各地[@shamshiri2018advances]。

以色列由于其独特的地理和气候条件，在节水灌溉和精准农业方面具有世界领先的技术水平。以色列的智能温室系统普遍采用滴灌和微灌技术，结合土壤湿度传感器和气象站数据，实现灌溉的精确控制。以色列的耐特菲姆（Netafim）公司是全球最大的滴灌设备制造商，其开发的智能灌溉系统能够根据作物需水量和土壤水分状况自动调节灌溉量，节水效率高达 50%以上[@morchid2024irrigation]。

日本在设施农业自动化方面具有丰富的经验，其智能温室系统注重机器人技术和人工智能的应用。日本国立农业和食品研究组织（NARO）开发的温室机器人能够自动完成播种、移栽、采摘、分级等作业，显著提高了劳动生产率。此外，日本还在温室环境控制中广泛应用模糊控制、神经网络等智能算法，实现了环境参数的优化调节[@bagagiolo2022greenhouse]。

美国在智能温室领域的研究侧重于大数据分析和云计算技术的应用。美国的 AgriTech 公司通过部署大量的物联网传感器，采集温室环境数据、作物生长数据和市场数据，利用机器学习算法进行分析和预测，为温室管理者提供决策支持。此外，美国还在垂直农业和植物工厂方面进行了积极探索，通过多层立体种植和人工光源技术，实现了单位面积产量的最大化[@shamshiri2013review]。

### 1.2.2 国内研究现状

我国智能温室技术的研究起步于 20 世纪 90 年代，经过近 30 年的发展，已经取得了显著的进步。特别是进入 21 世纪以来，随着物联网、大数据、人工智能等新一代信息技术的快速发展，我国智能温室技术进入了快速发展阶段。

在基础研究方面，中国农业大学、浙江大学、华南农业大学等高校在温室环境建模、控制算法优化、传感器技术等方面开展了大量研究工作。例如，中国农业大学的李振卿等学者对温室环境控制系统的硬件和软件设计进行了系统研究，提出了基于模糊控制和神经网络的智能控制算法，有效提高了温室环境控制的精度和稳定性[@liu2015intelligent]。浙江大学的王迎洁等学者对温室监控系统的关键技术进行了深入研究，开发了基于无线传感器网络的温室环境监测系统，实现了温室环境参数的实时采集和远程传输。

在技术应用方面，我国已经成功开发了多种类型的智能温室系统，广泛应用于蔬菜种植、花卉培育、果树栽培等领域。例如，中国农业科学院开发的"温室娃娃"智能温室管理系统，集成了温度、湿度、光照、土壤水分等多种传感器，支持手动控制、定时控制和智能控制三种工作模式，已经在全国多个省市推广应用。此外，华为、阿里云等科技企业也积极布局智慧农业领域，推出了基于云平台的智能温室解决方案，为温室管理者提供数据存储、分析和可视化服务。

然而，与发达国家相比，我国智能温室技术仍存在一些差距和不足。首先，在核心传感器和执行器方面，我国仍依赖进口，导致系统成本较高。其次，在控制算法方面，我国的研究主要集中在传统 PID 控制和模糊控制，对先进控制算法如模型预测控制（MPC）、自适应控制等的研究和应用还不够深入。最后，在系统集成和标准化方面，我国智能温室系统的兼容性和互操作性还有待提高。

### 1.2.3 智能温室系统的关键技术

智能温室系统涉及多个技术领域，其关键技术主要包括以下几个方面：

第一，传感器技术。传感器是智能温室系统的"感官"，负责采集温室环境参数。常用的传感器包括温湿度传感器（如 SHT30、DHT22）、光照传感器（如 BH1750、TSL2561）、土壤湿度传感器、二氧化碳浓度传感器等。传感器的精度、响应速度和稳定性直接影响系统的控制效果。近年来，微机电系统（MEMS）技术和纳米技术的发展，使得传感器的体积更小、功耗更低、精度更高，为智能温室系统的微型化和低成本化提供了可能[@sht30datasheet; @bh1750datasheet]。

第二，控制算法。控制算法是智能温室系统的"大脑"，负责根据采集到的环境参数和预设的目标值，生成控制指令。常用的控制算法包括开关控制（迟滞控制）、PID 控制、模糊控制、神经网络控制等。其中，PID 控制算法因其结构简单、参数整定方便、鲁棒性好等优点，在温室环境控制中得到了广泛应用。然而，由于温室环境具有非线性、时变、大滞后等特点，传统 PID 控制难以取得理想的控制效果。因此，近年来学者们提出了许多改进的 PID 控制算法，如自适应 PID 控制、模糊 PID 控制、神经网络 PID 控制等[@ang2005pid; @zeng2012nonlinear]。

第三，通信技术。通信技术是智能温室系统的"神经"，负责连接各个子系统和设备。常用的通信技术包括有线通信（如 RS485、CAN 总线）和无线通信（如 WiFi、ZigBee、LoRa）。其中，CAN 总线作为一种成熟的现场总线技术，具有高可靠性、实时性和灵活性等优点，被广泛应用于汽车电子、工业控制等领域。在智能温室系统中，CAN 总线可以用于连接主控制器和各个传感器节点、执行器节点，实现数据的可靠传输和设备的协同控制。

第四，人机交互技术。人机交互技术是智能温室系统的"面孔"，负责向用户展示系统状态和接收用户指令。常用的人机交互方式包括触摸屏显示、手机 APP、Web 网页等。其中，嵌入式触摸屏显示因其成本低、响应快、易于集成等优点，在智能温室系统中得到了广泛应用。LVGL 是一个开源的嵌入式 GUI 库，支持多种显示控制器和输入设备，能够提供丰富的界面元素和流畅的用户体验，非常适合应用于资源受限的嵌入式平台。

### 1.2.4 现有系统的不足与本课题的创新点

通过对国内外研究现状的分析，可以发现现有智能温室系统存在以下不足：

首先，系统架构方面。现有系统多采用单处理器架构，即由一个微控制器同时负责数据采集、控制算法执行、通信和人机交互等所有任务。这种架构在系统功能简单时能够满足要求，但随着系统功能的不断增加，单处理器的处理能力逐渐成为瓶颈，导致系统响应速度变慢、控制精度下降。

其次，控制算法方面。现有系统多采用传统 PID 控制或开关控制，这些算法虽然实现简单，但难以适应温室环境的非线性、时变和大滞后特性，导致控制效果不理想。此外，现有系统多采用单一的控制算法，缺乏根据环境变化自动切换控制策略的能力。

最后，人机交互方面。现有系统的人机交互界面多为简单的数值显示和按钮操作，缺乏直观的图形化展示和智能化的交互方式。用户需要具备一定的专业知识才能正确操作系统，降低了系统的易用性。

针对上述不足，本课题在系统架构、控制算法和人机交互三个方面进行了创新：

第一，采用 ESP32 和 STM32 双处理器架构。本系统将人机交互、网络通信等复杂功能分配给 ESP32 处理器，将实时性要求高的传感器数据采集和控制算法执行分配给 STM32 处理器，通过 CAN 总线实现两个处理器之间的数据交换和协同工作。这种分工协作的架构设计，既发挥了 ESP32 在网络通信和人机交互方面的优势，又充分利用了 STM32 在实时控制方面的特长，实现了系统性能的最优化。

第二，采用分层控制策略。本系统根据被控对象的不同特性，采用不同的控制算法：对于温度、湿度、土壤水分等变化较慢的参数，采用迟滞控制（开关控制）；对于通风风扇的转速控制，采用位置式 PID 控制算法。这种分层控制策略既保证了控制效果，又降低了系统复杂度。

第三，集成 AI 智能助手。本系统通过集成 DeepSeek 大语言模型 API，实现了基于自然语言交互的智能温室管理。用户可以通过语音或文字与 AI 助手对话，查询温室环境数据、设置控制参数、获取种植建议等，大大提高了系统的易用性和智能化水平。

## 1.3 论文主要研究内容

本课题围绕智能温室系统的设计与实现，开展了以下几个方面的研究工作：

### 1.3.1 系统总体架构设计

本课题首先对智能温室系统的总体架构进行了设计。系统采用 ESP32 和 STM32 双处理器架构，分为交互层和控制层两个层次。交互层以 ESP32 为核心，负责人机交互、网络通信、AI 智能助手等功能；控制层以 STM32 为核心，负责传感器数据采集、控制算法执行、执行器驱动等功能。两个层次之间通过 CAN 总线进行数据交换和协同工作。

在硬件架构方面，系统主要包括以下模块：ESP32 最小系统、STM32 最小系统、CAN 收发器模块、温湿度传感器模块（SHT30）、光照传感器模块（BH1750）、土壤湿度传感器模块、通风风扇模块、加湿器模块、水泵模块、LED 补光灯模块、OLED 显示模块等。这些模块通过合理的电路设计和布局，构成了一个完整的智能温室硬件平台。

在软件架构方面，系统采用分层设计思想，将软件分为驱动层、服务层和应用层三个层次。驱动层负责硬件设备的初始化和基本操作；服务层负责提供通用的功能服务，如通信协议栈、传感器数据管理、GUI 管理等；应用层负责实现具体的业务逻辑，如环境监测、自动控制、AI 交互等。这种分层设计提高了软件的可维护性和可扩展性。

### 1.3.2 STM32 控制层软件设计

STM32 控制层是智能温室系统的核心，负责传感器数据采集、控制算法执行和执行器驱动。本课题在 STM32 控制层的软件设计方面开展了以下工作：

第一，传感器驱动开发。本课题为 SHT30 温湿度传感器和 BH1750 光照传感器开发了基于 I2C 总线的驱动程序。SHT30 驱动程序实现了传感器的初始化、单次测量、周期性测量等功能，能够提供高精度的温度和湿度数据。BH1750 驱动程序实现了传感器的初始化、连续测量、单次测量等功能，能够提供高精度的光照强度数据。这些驱动程序采用了 Embassy 异步框架，能够有效利用 STM32 的硬件资源，提高系统的实时性。

第二，PID 控制算法实现。本课题实现了一个离散位置式 PID 控制器，用于通风风扇的转速闭环控制。PID 控制器的结构体定义如下：

```rust
pub struct PidController {
    integral: f32,          // 积分累积量
    prev_error: f32,        // 上一次误差
    integral_limit: f32,    // 积分限幅值
    output_limit: f32,      // 输出限幅值
    last_error: f32,        // 最近一次误差
    last_derivative: f32,   // 最近一次微分值
    last_output: f32,       // 最近一次输出值
}
```

该 PID 控制器的 `compute` 方法接收设定值、测量值、时间步长和比例、积分、微分三个参数，计算控制输出。在计算过程中，采用了积分限幅（Anti-windup）和输出限幅策略，防止积分饱和和执行器过载。

第三，自动调度器设计。本课题设计了一个基于迟滞控制的自动调度器，用于协调各个执行器的工作。自动调度器根据当前环境参数和目标值，采用迟滞控制策略决定各个执行器的开关状态。例如，对于温度控制，当温度高于目标值加上 `TEMP_FAN_ON_MARGIN_C`（1.0°C）时开启风扇，当温度低于目标值加上 `TEMP_FAN_OFF_MARGIN_C`（0.3°C）时关闭风扇。这种迟滞控制策略能够有效避免执行器的频繁开关，延长设备寿命。

第四，CAN 协议实现。本课题实现了一个自定义的 CAN 通信协议，用于 ESP32 和 STM32 之间的数据交换。协议采用 11 位标准帧格式，帧 ID 由 4 位功能码和 7 位节点 ID 组成。功能码定义了四种类型：告警（Alert）、时间同步（TimeSync）、写入（Write）和报告（Report）。数据域采用 8 字节固定长度，包含参数索引和参数值两个字段。该协议支持广播和点对点两种通信方式，能够满足智能温室系统的通信需求。

### 1.3.3 ESP32 交互层软件设计

ESP32 交互层负责智能温室系统的人机交互、网络通信和 AI 智能助手等功能。本课题在 ESP32 交互层的软件设计方面开展了以下工作：

第一，LVGL 图形界面开发。本课题基于 LVGL 图形库开发了一套完整的智能温室监控界面。界面采用多页面设计，包括主页、总览页、控制页、趋势图页、设置页、手动模式页、自动模式页和 AI 飞行员模式页。每个页面都有特定的功能，例如主页显示当前时间和系统状态，总览页显示所有传感器数据和执行器状态，控制页提供手动控制和自动控制的切换按钮等。LVGL 界面采用了 Guider 工具进行可视化设计，提高了开发效率。

第二，WiFi 网络连接。本课题实现了 ESP32 的 WiFi 连接管理功能，支持 STA（Station）和 AP（Access Point）两种模式。在 STA 模式下，ESP32 连接到现有的无线网络，实现互联网接入；在 AP 模式下，ESP32 自身创建一个无线热点，允许其他设备直接连接。WiFi 连接管理采用了自动重连机制，确保网络连接的稳定性。

第三，DeepSeek API 集成。本课题集成了 DeepSeek 大语言模型 API，实现了 AI 智能助手功能。DeepSeek API 采用了 HTTP RESTful 接口，支持流式响应，能够实时返回 AI 生成的内容。AI 智能助手能够理解用户的自然语言指令，查询温室环境数据、设置控制参数、提供种植建议等。为了提高用户体验，本课题还实现了语音识别和语音合成功能，用户可以通过语音与 AI 助手进行交互。

第四，百度语音 API 集成。本课题集成了百度语音 API，实现了语音识别和语音合成功能。语音识别功能将用户的语音指令转换为文字，供 DeepSeek API 处理；语音合成功能将 AI 助手的文字回复转换为语音，通过扬声器播放。为了保证语音识别的准确性，本课题采用了端点检测（VAD）技术，自动识别语音的起始和结束位置。

### 1.3.4 系统测试与验证

本课题对智能温室系统进行了全面的测试和验证，包括功能测试、性能测试和稳定性测试。功能测试验证了系统各个模块的功能是否正常，包括传感器数据采集、控制算法执行、执行器驱动、CAN 通信、LVGL 界面显示、WiFi 连接、AI 智能助手等。性能测试评估了系统的响应速度、控制精度和资源占用情况。稳定性测试验证了系统在长时间运行下的可靠性。

测试结果表明，本课题设计的智能温室系统能够满足设计要求，实现了温室环境的自动监测和控制，具有较高的实用价值和推广前景。
