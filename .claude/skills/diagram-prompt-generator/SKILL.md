# 画图提示词生成器

## 概述

本技能用于生成给画图 AI（如 Midjourney、DALL-E、Stable Diffusion 等）的提示词。输出的提示词会自动包含底部留白要求，方便后期裁剪去除水印。

## 何时使用本技能

- 用户需要生成画图 AI 的提示词。
- 用户描述了一个图表/框图/示意图的需求。
- 用户要求"生成画图提示词"、"diagram prompt"、"画图提示词"。

## 何时不使用本技能

- 用户需要绘制 Mermaid 图表。请改用 `mermaid-renderer`。
- 用户需要直接编写代码生成图表。

## 核心步骤

### 步骤 1：提取用户需求

从用户输入中提取以下信息：

| 要素 | 说明 | 示例 |
|---|---|---|
| 图表类型 | 框图、流程图、架构图、示意图等 | 硬件总体框图 |
| 主要模块 | 图中需要包含的模块/组件 | ESP32、STM32、传感器 |
| 连接关系 | 模块之间的连接方式 | CAN 总线、I2C、WiFi |
| 布局偏好 | 左右、上下、中心辐射等 | 左侧主节点，右侧从节点 |
| 风格要求 | 黑白、彩色、工程风、卡通等 | 黑白工程图 |

### 步骤 2：生成提示词结构

提示词应包含以下部分：

```
1. 开头：图表类型 + 风格要求
2. 主体：各模块及其包含的子组件
3. 连接：模块间的连接关系和标注
4. 布局：整体排列建议
5. 画布要求：底部留白（固定模板）
```

### 步骤 3：底部留白模板

**所有输出的提示词必须包含以下画布要求：**

```
**画布要求**：图片底部请预留大量空白区域（约占画布高度的 20%-25%），所有硬件框图内容集中在画布上方 75% 区域内，底部空白用于放置标识水印，方便后期裁剪。
```

### 步骤 4：输出格式

输出的提示词应为引用块格式，用户可直接复制使用：

```
> 请绘制一张[图表类型]，风格为[风格要求]。
>
> **[模块1名称]**包含：[子组件列表]
>
> **[模块2名称]**包含：[子组件列表]
>
> **连接**：[连接关系说明]
>
> 布局建议：[布局说明]
>
> **画布要求**：图片底部请预留大量空白区域（约占画布高度的 20%-25%），所有硬件框图内容集中在画布上方 75% 区域内，底部空白用于放置标识水印，方便后期裁剪。
```

## 示例

### 输入

用户：帮我生成一个提示词，画一个智能温室系统的硬件框图，有ESP32主节点和STM32从节点，通过CAN总线连接。

### 输出

> 请绘制一张智能温室系统的硬件总体框图，风格为黑白工程图。
>
> **主节点**（ESP32-S3）包含：触摸屏、麦克风(INMP441)、功放(MAX98357A)、WiFi模块、CAN收发器(TJA1051T)
>
> **从节点**（STM32F407VET6）包含：温湿度传感器(SHT30)、光照传感器(BH1750)、土壤湿度传感器(ADC)、风扇(TB6612)、水泵(继电器)、加湿器(继电器)、补光灯(WS2812B)、舵机(MG995)、CAN收发器(TJA1051T)
>
> **连接**：主节点与从节点通过 CAN 总线互联，总线两端标注 120Ω 终端电阻。
>
> 布局建议：左侧为主节点，右侧为从节点，中间为 CAN 总线，传感器在从节点上方，执行器在下方。
>
> **画布要求**：图片底部请预留大量空白区域（约占画布高度的 20%-25%），所有硬件框图内容集中在画布上方 75% 区域内，底部空白用于放置标识水印，方便后期裁剪。

## 注意事项

1. 提示词语言与用户输入语言保持一致（中文输入 → 中文提示词）
2. 如果用户没有指定风格，默认使用"黑白工程图"
3. 如果用户没有指定布局，根据图表类型给出合理建议
4. 底部留白模板为固定内容，不可省略
