# 论文规范符合性检查报告

## 检查依据

`Docs/chapters/论文要求/本科毕业设计（论文）要求与撰写规范（ 理工类）-本部潇湘 (1).md`

## 检查范围

`Docs/chapters/` 目录下的 6 个章节 Markdown 文件及 `Docs/references.bib`。

## 一、结构完整性 — 严重缺失

### 1.1 缺失的组成部分

规范要求毕业论文包含**前置部分、主体部分、结尾部分**三大部分，当前仅完成了主体部分的 6 章正文。以下部分完全缺失：

| 缺失部分 | 规范要求 | 严重程度 |
|:---|:---|:---|
| **封面** | 单独一页，题目不超过 20 个汉字 | 严重 |
| **中文摘要及关键词** | 摘要独立短文，3–5 个关键词用分号隔开 | 严重 |
| **英文摘要及关键词** | ABSTRACT + Keywords，内容须与中文摘要一致 | 严重 |
| **目录** | 自动生成，含页眉"湖南科技大学本科生毕业设计（论文）" | 严重 |
| **参考文献列表** | 正文后独立成章，GB/T 7714-2015 格式 | 严重 |
| **附录** | 程序代码、实验数据等，标识为"附录A""附录B" | 中等 |
| **致谢** | 致谢校内外指导教师及帮助者 | 中等 |

> **说明**：虽然有 `references.bib` 文件供 Pandoc 生成参考文献，但当前各章节 Markdown 中没有包含参考文献输出的章节文件，且缺少摘要、目录、封面、附录、致谢的 Markdown 源文件。

### 1.2 结论章独立性问题

规范 §2.3.3 明确要求"结论"部分应设为**独立的一章**。当前 Chapter6 标题为"第6章 总结与展望"，将"总结"与"展望"合并在同一章中，虽然结构上可行，但需确认学校是否接受"总结与展望"作为章标题——部分院校要求章标题仅写"结论"。

## 二、章节编号格式问题

### 2.1 章标题编号应使用大写数字

规范 §4.3 明确规定：

> "章编号用大写数字"，示例为"第三章  ※※…※"

当前所有章标题使用的都是阿拉伯数字：

- `# 第1章 绪论` → 应为 `# 第一章 绪论`
- `# 第2章 需求分析与总体设计` → 应为 `# 第二章 需求分析与总体设计`
- `# 第3章 硬件电路设计` → 应为 `# 第三章 硬件电路设计`
- `# 第4章 STM32控制层软件设计` → 应为 `# 第四章 STM32控制层软件设计`
- `# 第5章 ESP32交互层软件设计` → 应为 `# 第五章 ESP32交互层软件设计`
- `# 第6章 总结与展望` → 应为 `# 第六章 总结与展望`

**严重程度**：中等（Pandoc + Word 模板可能在最终编排时修正，但 Markdown 源文件应遵循规范）

### 2.2 节、条编号使用阿拉伯数字 — 已正确

当前节编号格式如"1.1""1.2.1"符合规范要求"X.1""X.1.1"的格式。

### 2.3 正文层次过深

规范 §3.2 要求"正文层次以少为宜，一般不超过三级"（即章、节、条三级）。当前部分章节使用了四级：

- Chapter4: `#` → `##` → `###` → 正文中的 `（1）` 款级（可接受）
- 但 Chapter1 中 `##` 下有 `###` 再加 `（1）` 载项 — 实际已到四级层次

需确认是否符合学校实际接受标准。

## 三、公式编排问题

### 3.1 公式缺少编号（严重）

规范 §4.4 要求：

> "公式序号按章独立编号。例如：第一章第1个公式序号为'（1.1）'"

当前所有公式均**没有编号**：

| 文件 | 公式位置 | 现状 | 应为 |
|:---|:---|:---|:---|
| Chapter4 §4.2.1 | SHT30 温度转换公式 | 无编号 | (4.1) |
| Chapter4 §4.2.1 | SHT30 湿度转换公式 | 无编号 | (4.2) |
| Chapter4 §4.2.3 | 土壤湿度映射公式 | 无编号 | (4.3) |
| Chapter4 §4.3.1 | PID 输出公式 | 无编号 | (4.4) |
| Chapter2 §2.2.2 | CAN_ID 构建公式（行内） | 无编号 | (2.1) |

### 3.2 公式引用不规范

规范 §4.4 要求正文中引用公式时应使用"见式（X.Y）"或"由公式（X.Y）"等规范表述。当前正文中没有任何对公式的规范引用。

### 3.3 公式中物理量缺少首次解释

规范 §4.4 要求"公式中首次出现的物理量符号应予以解释说明"。

- Chapter4 PID 公式：$K_p$、$K_i$、$K_d$、$\Delta t$ 等参数虽有部分解释，但未在公式后集中列出符号说明
- Chapter4 SHT30 公式：$S_T$ 和 $S_{RH}$ 在公式后有说明 — 已基本满足
- Chapter4 土壤湿度公式：$D_{ADC}$ 有说明但 4000 和 1000 仅文字描述，未明确为"校准常数"

## 五、插图编排问题

### 5.1 图题位置

规范 §4.6 要求图题"整体置于图的下方"。当前所有图题均写在图片**上方**（先 `**图 X-X 标题**`，再 `![...]()`），例如：

```markdown
**图 2-1 分布式多节点系统总体架构**

![图 2-1 分布式多节点系统总体架构](../images/ai_generated/cropped/system_architecture.png)
```

应改为图题在图片下方。不过当前格式在 Pandoc 转换时，如果使用自定义模板可将图片标题提取到下方。需在最终排版中确认。

### 5.2 图片引用顺序问题

规范要求"正文中必须先引用图表，再出现图表"。经逐一检查，所有图表均在正文叙述中先引用后出现 — 合规。

### 5.3 图片路径双斜杠问题

多个文件中图片路径包含双斜杠 `//`：

- Chapter4: 多处 `../images/ai_generated/cropped//stm32_task_overview.png`
- Chapter5: 多处 `../images/ai_generated/cropped//esp32_task_flow.png`
- Chapter3: 图 3-5、图 3-6

虽然多数操作系统能容忍双斜杠，但这属于路径错误，可能导致部分工具解析失败。

## 六、引用文献问题

### 6.1 标题处引用

规范 §3.3 明确要求"禁止在各级标题处进行文献标注"。当前各章标题中未发现引用标注 — 合规。

### 6.2 引用格式

当前使用 `[@key]` 格式（Pandoc citeproc 语法），规范要求"顺序编码制"——Pandoc 配合 CSL 文件 `china-national-standard-gb-t-7714-2015-numeric.csl` 可自动生成 `[1]` 格式。格式本身合规。

### 6.3 连续引用写法

部分连续引用使用了 `[@key1; @key2]` 格式（如 Chapter1 §1.2.1 中），这是 Pandoc 的正确语法，citeproc 会自动合并为 `[x, y]`。合规。

### 6.4 数据手册引用类型

规范 §4.7 的参考文献类型列表中没有 `@manual` 类型，但 GB/T 7714-2015 中数据手册通常归类为"技术报告[R]"或"标准[S]"。当前 `references.bib` 中使用 `@manual` 类型：

- `sht30datasheet`、`bh1750datasheet`、`esp32techref`、`stm32f407datasheet`、`tja1051datasheet`、`inmp441datasheet`、`max98357datasheet`

建议将 `@manual` 改为 `@techreport` 或保持现状由 CSL 自动映射。需测试 Pandoc 输出结果确认格式是否正确。

### 6.5 在线资源引用类型

`references.bib` 中使用 `@online` 类型（`lvgl2024`、`embassy2024`、`embeddedhal2024`），BibTeX 标准中没有 `@online` 类型，但 Pandoc citeproc 支持。需确认 CSL 输出是否符合 GB/T 7714-2015 的电子文献格式要求。

## 七、段落与文字格式问题

### 7.1 条目标题格式不一致

规范 §4.2 要求条级标题（X.X.X）使用"小四号宋体加粗，段前段后0.5行"。当前格式为：

- Chapter1 中 `### 1.2.1 国外研究现状` 等 — 使用 Markdown `###` 语法，Pandoc 会映射为三级标题 — 合规
- 但 Chapter1 中 `（1）` `（2）` 等款级标题使用了 `**粗体**` 标记，如：


  规范要求款、项标题为"小四号宋体加粗，无段间距，1.25倍行距，首行缩进2字符"。当前写法在 Pandoc 转换中可能无法正确映射为款级标题样式。
